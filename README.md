# korean-business-writing-skill

한국어 비즈니스 문서의 품질을 개선하는 Claude Code 스킬입니다.
Claude Code는 Anthropic의 명령줄 코딩 도구이고, 스킬은 특정 작업에서만 로드되는 확장 단위입니다.

## 문제 정의

LLM은 분량은 채우지만 완성도가 낮은 글을 씁니다.
특히 다음 세 가지 문제가 반복됩니다.

**구조 문제.** 결론이 뒤에 배치되고 항목 간 위계가 정리되지 않습니다.

**독자 고려 문제.** 자신이 아는 전문용어와 줄임말을 독자도 안다고 전제합니다.
설명 수준을 독자가 아닌 자신에게 맞춥니다.

**자기 검토의 한계.** 작성자는 자기 글의 결함을 보지 못합니다.
지침을 알더라도 스스로 점검하면 같은 맹점을 반복합니다.

앞의 두 문제는 작성 지침으로, 마지막 문제는 검토 루프로 해결합니다.

## 구성

- 작성 지침 (`SKILL.md`) — 글을 쓰거나 다듬을 때 적용합니다.
- 검토 루프 (`loop.md`, `reviewers/`) — 글을 완성한 뒤 검증합니다.

## 적용 범위

비문학 중 비즈니스 글쓰기를 대상으로 합니다.

- 기획서
- 계획서
- 매뉴얼
- 설명 문서
- 이력서

코드, 커밋 메시지, 기술 명세에는 적용하지 않습니다.

한국어 문서를 전제로 합니다. 종결어미 통일과 가운데점 금지 등
한국어 표기에 한정된 지침을 포함합니다.

## 작성 지침

### 구조화 지침

- 두괄식(결론을 앞에 두는 구성)으로 작성합니다.
- 간결하게 작성합니다.
- 내용이 많아지면 구조화합니다.
- 상위 항목과 하위 항목의 차원을 일치시킵니다.
- 문장 종결어미를 항목의 성격에 맞춥니다.

### 독자 수준 고려 지침

- 전문용어와 줄임말을 독자가 안다고 가정하지 않습니다.
- 그렇다고 전부 설명하지 않습니다. 독자 수준에 맞춰 생략할 부분을 판단합니다.
- 맥락을 모르는 사람이 읽어도 이해할 수 있어야 합니다.
- 전문용어는 설명한 다음에 사용합니다.

### 표기 지침

- 가운데점(·)을 사용하지 않습니다.
- 구어체와 비속어를 사용하지 않습니다.

전체 지침은 [SKILL.md](SKILL.md)를 참고하십시오.

## 검토 루프

검토 루프는 작성자가 아닌 별도 관점이 완성된 글을 검증하는 절차입니다.
관점이 다른 세 검토자를 두고, 각 검토자는 서로 겹치지 않는 실패 유형을 하나씩 맡습니다.

- 맥락 전문가 A — 장황함을 검출합니다. 중복되거나 삭제해도 되는 문장을 잡습니다.
- 무맥락 독자 B — 설명 부족을 검출합니다. 배경을 모르면 이해할 수 없는 대목을 잡습니다.
- 표기와 구조 감사관 C — 규칙 위반을 검출합니다. 작성 지침(SKILL.md)을 어긴 지점을 잡습니다.

한 검토자라도 통과시키지 못하면 지적사항을 반영해 다시 검증합니다.
세 검토자를 모두 통과할 때까지 반복하되, 최대 3회까지 하고
그래도 통과하지 못하거나 검토자의 요구가 상충하면 사람이 판단합니다.

- `loop.md` — 검증 절차를 정의합니다.
- `reviewers/` — 세 검토자의 설정과 평가 기준을 파일별로 담습니다.

## 왜 스킬로 분리했는가

전역 설정(CLAUDE.md)은 모든 요청마다 로드되어 코드 작업에도 글쓰기 지침이 끼어듭니다.
스킬은 그렇지 않으므로, 글쓰기 지침을 담기에 적합합니다.

## 설치

### 방법 1. 심볼릭 링크

저장소를 clone한 뒤 링크를 연결합니다.
저장소에서 파일을 수정하면 스킬에 즉시 반영됩니다.

```bash
git clone https://github.com/사용자명/korean-business-writing-skill.git ~/projects/korean-business-writing-skill
ln -s ~/projects/korean-business-writing-skill ~/.claude/skills/business-writing
```

### 방법 2. 직접 복사

```bash
git clone https://github.com/사용자명/korean-business-writing-skill.git
cp -r korean-business-writing-skill ~/.claude/skills/business-writing
```

두 방법 모두 스킬명을 `business-writing`으로 지정합니다.
Claude Code는 디렉터리명을 스킬명으로 인식하므로,
저장소명을 그대로 사용하면 호출 명령어가 불필요하게 길어집니다.

설치 후 Claude Code를 재시작하십시오. 새 스킬은 세션 시작 시 로드됩니다.

## 사용법

### 작성 지침 적용

글쓰기 작업을 요청하면 Claude가 이 스킬을 자동으로 불러 적용합니다.
명시적으로 지정하려면 `/business-writing`을 호출합니다.

### 검토 루프 실행

글을 완성한 뒤 `loop.md`의 절차에 따라 검증합니다.

## 한계

작성 지침과 검토 루프를 함께 적용해도 최종 검수는 여전히 필요합니다.

## 향후 계획

문서 유형별로 검토자 구성을 고르는 프로파일 기능을 검토 중입니다.
