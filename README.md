# korean-business-writing-skill

한국어 비즈니스 문서 작성 품질을 개선하는 Claude Code 스킬입니다.

## 문제 정의

LLM은 글을 잘 쓰지 못합니다. 분량은 생성하지만 결과물의 완성도가 낮습니다.
특히 다음 두 가지 문제가 반복됩니다.

**구조 문제.** 결론이 뒤에 배치되고 항목 간 위계가 정리되지 않습니다.

**독자 고려 문제.** 자신이 아는 전문용어와 줄임말을 독자도 안다고 전제합니다.
설명 수준을 독자가 아닌 자신에게 맞춥니다. 비유를 요청해야 비로소 쉽게 설명합니다.

이 스킬은 두 문제를 해결하기 위한 지침을 제공합니다.

## 왜 스킬로 분리했는가

전역 설정 파일(CLAUDE.md)은 모든 요청마다 로드됩니다.
글쓰기 지침을 여기에 넣으면 코드 작업 시에도 적용되어 방해가 됩니다.

스킬은 관련 작업에서만 로드됩니다.
따라서 글쓰기 지침은 스킬로 분리하는 것이 적합합니다.

## 적용 범위

비문학 중 비즈니스 글쓰기를 대상으로 합니다.

- 기획서
- 계획서
- 매뉴얼
- 설명 문서
- 이력서

코드, 커밋 메시지, 기술 명세에는 적용하지 않습니다.

한국어 문서를 전제로 합니다. 어말어미 통일과 가운데점 금지 등
한국어 표기에 한정된 지침을 포함합니다.

## 지침 구성

### 구조화 지침

- 두괄식으로 작성한다.
- 간결하게 작성한다.
- 내용이 많아지면 구조화한다.
- 상위 항목과 하위 항목의 차원을 일치시킨다.
- 문장 종결어미를 항목의 성격에 맞춘다.

### 독자 수준 고려 지침

- 전문용어와 줄임말을 독자가 안다고 가정하지 않는다.
- 그렇다고 전부 설명하지 않는다. 독자 수준에 맞춰 생략할 부분을 판단한다.
- 맥락을 모르는 사람이 읽어도 이해할 수 있어야 한다.
- 전문용어는 설명한 다음에 사용한다.

### 표기 지침

- 가운데점을 사용하지 않는다.
- 구어체와 비속어를 사용하지 않는다.

전체 지침은 [SKILL.md](SKILL.md)를 참고하십시오.

## 설치

### 방법 1. 심볼릭 링크

저장소를 clone한 뒤 링크를 연결합니다.
저장소에서 파일을 수정하면 스킬에 즉시 반영됩니다.

```bash
git clone https://github.com/사용자명/korean-business-writing-skill.git ~/projects/korean-business-writing-skill
ln -s ~/projects/korean-business-writing-skill ~/.claude/skills/business-writing
```

### 방법 2. 직접 복사

```bash
git clone https://github.com/사용자명/korean-business-writing-skill.git
cp -r korean-business-writing-skill ~/.claude/skills/business-writing
```

두 방법 모두 스킬명을 `business-writing`으로 지정합니다.
Claude Code는 디렉터리명을 스킬명으로 인식하므로,
저장소명을 그대로 사용하면 호출 명령어가 불필요하게 길어집니다.

설치 후 Claude Code를 재시작하십시오. 새 스킬은 세션 시작 시 로드됩니다.
