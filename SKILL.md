# 비즈니스 글쓰기 지침

## 대원칙
1. 두괄식으로 작성한다.
2. 간결하게 작성한다.
3. 내용이 많아지면 구조화한다.

## 구조화 방법
1. 글의 구성을 항목별로 쪼갠다.
2. 항목 내에서도 하위 항목으로 쪼갠다.
3. 하위 항목들은 논리적 공통점이 있는 상위 항목으로 묶는다.
4. **하위 항목들은 차원이 맞아야 한다.**
   - 예: '대응 방안'이 상위 항목이면 하위는 방안들로만 구성한다.
     기술 항목이나 문제점이 섞이면 안 된다.
5. **문장 종결도 차원에 맞춘다.**
   - 예: '이벤트 운영방안'의 하위 항목이라면 "예약 웹페이지 운영"(O),
     "예약 웹페이지 배포함"(X)
   - '배포함'은 이미 완료했다는 의미를 포함하므로 '방안'으로 볼 수 없다.
     다만 모호한 경우가 있으므로 적절히 판단한다.
6. 어말어미와 종결어미를 상황에 맞게 사용하며 가능한 한 통일한다.
   (예: `~임.`으로 통일하거나 `~절감.`처럼 명사형으로 통일)

## 독자 고려
7. 내가 아는 전문용어나 줄임말을 남도 안다고 가정하지 않는다.
8. 그렇다고 처음부터 끝까지 전부 설명하지는 않는다.
   독자 수준에 맞춰 생략할 부분을 판단하여 간결성을 유지한다.
9. 맥락을 전혀 모르는 사람이 읽어도 이해할 수 있어야 한다.
   "내가 이해하기 편한 수준"으로 남기지 않는다.
10. 전문용어는 설명한 다음에 사용한다.
    (예: "k 스윕이 무엇인지부터 설명하고 진행")

## 표기 규칙
11. **가운데점(·) 사용을 절대 금지한다.** 쉼표 또는 '와/과'를 사용한다.
    - 예: "설계·개발·배포"(X) → "설계, 개발, 배포"(O)
12. **구어체와 비속어를 금지한다.** 문어체로 작성한다.
    - 예: "이거 좀 애매함"(X) → "판단이 모호함"(O)

## 문장 공법
13. **한 문장에는 하나의 사건만 담는다.**
    독립적으로 존재할 수 있는 사건(행위와 그 결과)을 병렬 연결어미로 이어 붙이지 않는다.
    - 예: "A를 구축하고 B로 이관했으며, C를 새로 만들었다."(X)
      세 사건이 나열되면 각각의 무게가 사라진다. 문장을 나눈다.
    - 나열이 허용되는 경우는 같은 층위의 세부 요소일 때뿐이다.
      (예: "S3, Glue, Athena를 사용했다."(O) — 도구 목록은 하나의 사건)

14. **언급한 항목은 전개하거나 삭제한다.**
    한 절로만 스치고 더 설명되지 않는 항목은 독자에게 의문만 남긴다.
    - 중요하면 무엇을, 왜, 어떻게 했는지 한 단락으로 전개한다.
    - 중요하지 않으면 뺀다.
    - 어중간하게 걸쳐 두지 않는다.

15. **주어, 목적어, 서술어의 호응을 맞춘다.**
    "무엇을 → 어떻게 했다"가 실제로 성립하는지 문장마다 확인한다.
    - 예: "반복 업무를 자동화해 납품했다."(X)
      납품한 것은 업무가 아니라 산출물이다.
      → "업무를 자동화하는 도구를 만들어 납품했다."(O)

16. **두괄식의 첫 문장은 새 정보를 담아야 한다.**
    소제목이나 앞 문장을 재진술하는 첫 문장은 두괄식이 아니라 군더더기다.
    첫 문장에 결론과 함께 구체적 사실(대상, 수치, 결과)을 싣는다.
    - 예: 제목이 "[비용 절감 방안]"인데 첫 문장이 "비용을 절감했다"이면
      정보량이 0이다.

17. **서술 항목의 종결 명사를 의도에 맞게 쓴다.** 사물 명사로 끝내지 않는다.
    만든 것인지, 한 것인지, 할 수 있는 것인지를 판단해 종결어를 고른다.
    - 예: "…데이터 파이프라인"(X) → "…데이터 파이프라인 구축"(O)
    - 역량을 말할 때: "…분석"(X) → "…분석 경험"(O)

## 작성 전 확인
18. **항목의 비중을 모르면 쓰지 말고 확인한다.**
    무엇이 중요한지는 글쓴이의 의도와 맥락에 달려 있으므로 지침으로 정할 수 없다.
    각 항목의 중요도를 확인하고, 중요도에 비례해 분량을 배분한다.
    확인 없이 균등하게 나열하거나 떠오르는 순서대로 쓰지 않는다.

## 하지 말 것
- 미괄식으로 결론을 뒤에 숨기지 않는다.
- 상위 항목과 하위 항목의 차원을 섞지 않는다.
- 종결어미를 문서 내에서 뒤섞지 않는다.
- 가운데점을 사용하지 않는다.
- 구어체 표현을 사용하지 않는다.
- 독립된 사건을 한 문장에 나열하지 않는다.
- 언급만 하고 전개하지 않는 항목을 남기지 않는다.
- 목적어와 서술어의 호응을 깨지 않는다.
- 소제목을 재진술하는 첫 문장을 쓰지 않는다.
- 사물 명사로 서술 항목을 끝내지 않는다.
